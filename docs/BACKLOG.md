# Бэклог идей

Идеи, отложенные на потом (обсуждены 2026-07-14). Реализованное из того же
списка: ранжирование графиков по аномальности, таймлайн на главной,
материализованное представление почасовых агрегатов, панель
«как выполнился запрос» (X-ClickHouse-Summary), bloom-индекс для `event_id`, гео с картой,
кодеки сжатия (по замерам), TTL 90 дней (см. DATA_MODEL.md §6).

## Развитие кейса

- **Автопоиск аномалий** — команда или страница «подозрительные группы»:
  перебрать все пары измерение × метрика (один `GROUP BY` на пару), посчитать
  z-score отклонения каждой группы от глобального среднего, вывести топ.
  BubbleUp без выбора события: «у D-13 в 4 раза больше harsh_braking».

## Фишки ClickHouse

- **Прожекция для точечного поиска по `event_id`** — bloom-индекс уже добавлен;
  для сравнения можно попробовать `ADD PROJECTION ... ORDER BY event_id`
  (полная копия данных с другой сортировкой, ~двойной диск) и замерить разницу.
- **Кодеки сжатия** — `DoubleDelta` для `event_time`/счётчиков, `Gorilla` для
  float-метрик: `ALTER TABLE ... MODIFY COLUMN ... CODEC(...)`. Эффект виден
  в плитке «на диске» на главной до/после `OPTIMIZE TABLE ... FINAL`.
- **TTL и партиции** — `ALTER TABLE events MODIFY TTL event_time + INTERVAL 90 DAY`
  (автоудаление старого), мгновенный `ALTER TABLE events DROP PARTITION '202606'`.
  Жизненный цикл данных почти без кода.
- **Сэмплирование** — пересоздать таблицу с `SAMPLE BY` (ключ сэмплирования
  обязан входить в ORDER BY, т.е. полная перезаливка данных), графики по
  `SAMPLE 0.1` мгновенны на любом объёме ценой приблизительности. Отложено
  сознательно: кроме миграции таблицы нужна поддержка в API/UI (параметр
  sample, домножение счётчиков на 1/rate) — это не «быстрый ALTER».
